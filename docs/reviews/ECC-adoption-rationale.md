# ECC Adoption Rationale

Status: Approved

---

# Objective

Define the adoption policy for ECC.

---

# Adoption Strategy

ECC will not be installed as a complete framework.

Only reusable components with independent value will be incorporated into the knowledge base.

---

# Accepted Components

- High-value agents
- High-value skills
- Reusable MCP integrations
- Generic workflows
- Reusable engineering rules

---

# Rejected Components

- Complete framework installation
- Opinionated project structure
- Monolithic workflow
- Redundant automation
- Components already covered by Superpowers

---

# Selection Criteria

Every component must satisfy at least one of the following:

- Reusable
- Platform independent
- Well documented
- Maintainable
- Valuable outside ECC

---

# Architectural Role

Component Library
