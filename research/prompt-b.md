INVESTIGATION RULE

This task is not a repository summary.

It is a technical research process.

Before making any recommendation, investigate every repository and every documentation source as deeply as possible.

Do not rely exclusively on the README.

Whenever available, inspect:

- Documentation folders
- Wiki pages
- Examples
- Releases
- Configuration files
- CLI documentation
- Official websites
- Related repositories
- Linked documentation
- Community discussions when they provide technical value

The objective is to understand the complete ecosystem surrounding each resource.

Recommendations must be based on evidence gathered from all available sources, not only from repository descriptions.

If two repositories solve similar problems, compare them directly and recommend only the best option unless both clearly complement each other.

Avoid recommending redundant tools.

Prefer mature, actively maintained, well documented, reusable, provider-agnostic resources that integrate well with an AI development workflow.

OUTPUT RULE

Do not generate Markdown files for my repository.

Do not generate documentation intended to be copied directly into my knowledge base.

Your only objective is to produce a structured technical research report.

The report must contain:

- Executive summary
- Categorized recommendations
- Technical comparisons

- Advantages and disadvantages
- Recommended adoption order
- Resources rejected and reasons
- Missing capabilities detected in my current environment
- Final roadmap
