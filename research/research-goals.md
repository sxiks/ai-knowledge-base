# Research Goals

Version: v2.0

Last Updated: 2026-07-03

---

# Mission

Build a comprehensive, long-term knowledge base for AI-assisted software engineering.

The repository is intended to become the single source of truth for evaluating, documenting, comparing and adopting AI development resources across multiple platforms and providers.

---

# Primary Objectives

- Build a curated knowledge base instead of a collection of links.
- Preserve architectural decisions.
- Preserve technical evaluations.
- Preserve implementation knowledge.
- Preserve reusable workflows.
- Preserve engineering methodologies.
- Preserve platform-independent practices.

---

# Research Philosophy

Research must prioritize completeness before selection.

Every relevant resource should be investigated before deciding whether it deserves adoption.

The objective is to maximize knowledge acquisition while maintaining high documentation quality.

Recommendations are produced after research, never before.

---

# Long-Term Scope

The repository should continuously grow as new resources become available.

There are no predefined numerical limits for:

- Agents
- Subagents
- Skills
- MCP Servers
- Plugins
- Utilities
- Workflows
- Methodologies
- Platforms
- Engineering resources

Every resource that provides meaningful technical value should be documented.

---

# Quality Principles

Every documented resource should be evaluated according to:

- Purpose
- Architecture
- Maturity
- Community adoption
- Documentation quality
- Maintenance status
- Compatibility
- Reusability
- Advantages
- Limitations
- Learning curve
- Complexity
- Long-term viability

---

# Selection Policy

Research should be exhaustive.

Adoption should be selective.

Documentation should be comprehensive.

Recommendations should always be supported by technical evidence.

---

# Repository Objectives

The knowledge base should enable:

- Architecture reviews
- Technology comparisons
- Ecosystem evaluations
- Provider-independent workflows
- Knowledge preservation
- Future implementation planning
- Continuous ecosystem evolution

# Success Criteria

The repository should:

- Cover the AI engineering ecosystem as broadly as practical.
- Minimize duplicated knowledge.
- Preserve historical architectural decisions.
- Support multiple AI coding platforms.
- Remain maintainable as the ecosystem evolves.
- Serve as the authoritative reference for future implementation decisions.
