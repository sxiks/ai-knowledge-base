# Constraints

Recommendations should prioritize:

- Open Source
- Active maintenance
- Excellent documentation
- Provider independence
- Cross-platform compatibility
- Reusability
- Automation
- Long-term maintainability

Recommendations should avoid:

- Duplicate functionality
- Vendor lock-in
- Abandoned projects
- Unnecessary complexity
- Tools that overlap with already installed capabilities unless they provide a clear improvement.

---

# Research Focus

The objective is **not** to discover every available tool.

The objective is to build a high-quality AI ecosystem that:

- Integrates naturally with OpenCode.
- Can later support Claude Code, Codex CLI, Gemini CLI and Cursor.
- Is modular.
- Is reproducible.
- Can be automatically installed through a bootstrap process.
- Maximizes productivity while minimizing maintenance.

---

# Research Preferences

When multiple tools solve the same problem, recommendations should prioritize:

1. Reusable across multiple AI platforms.
2. Provider-agnostic solutions.
3. CLI-first tools.
4. Open-source projects.
5. Active maintenance.
6. Excellent documentation.
7. Modular architecture.
8. Easy automation.
9. Integration with Git-based workflows.
10. Long-term scalability.

Quality is preferred over quantity.

A smaller, well-integrated ecosystem is preferred over a large collection of overlapping tools.
